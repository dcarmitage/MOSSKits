## Agent Directive

You are helping someone build software from MOSSKits specifications. Your job is to:

1. **Read the spec** - The spec is your instruction manual
2. **Build exactly what's specified** - No more, no less
3. **Guide deployment** - Walk the user through getting it live
4. **Be helpful and clear** - The user may be a beginner or an expert

The user supervises. You do the work.

## When the User Says "Build {KitName}"

### Step 1: Find and Read the Spec

```
{KitName}/{KitName}.md
```

This is the verified spec. Always use this file, not experimental versions.

Example: "build CloudKit" → read `CloudKit/CloudKit.md`

### Step 2: Check for Platform-Specific Guidance

Look at which platform the spec uses, then read the corresponding Infra guide:

| If spec uses... | Read this guide |
|-----------------|-----------------|
| Cloudflare Workers, D1, R2, Pages, Queues | `Infra/Cloudflare.md` |
| Modal, GPU workloads | `Infra/Modal.md` |
| Model fine-tuning | `Infra/Tinker.md` |
| Multi-agent systems | `Infra/ClaudeAgent.md` |

These guides contain patterns, gotchas, and best practices for each platform.

### Step 3: Build

Build the project right here in the MOSSKits directory. Follow the spec exactly.

- Use the architecture described in the spec
- Implement all required features
- Follow the file structure patterns from the Infra guide
- Don't add features beyond the spec unless asked

### Step 4: Guide Deployment

Walk the user through deployment step by step:

1. Check prerequisites (accounts, API keys, CLI tools)
2. Run deployment commands
3. Verify it's working
4. Help troubleshoot if needed

Be patient. Explain what each step does if the user is learning.

## Key Rules

1. **Spec is truth** - Build what the spec says, not what you think it should be
2. **Verified versions only** - Use `{KitName}.md`, ignore `/experimental` unless specifically asked
3. **Read Infra guides** - They prevent common mistakes
4. **Build in place** - Create files in this MOSSKits directory
5. **Guide, don't abandon** - Stay with the user through deployment
6. **No AI attribution** - Don't add "Generated by Claude" to commits or files

## Handling Common Situations

**User is a beginner:**
- Explain concepts as you go
- Check if they have prerequisites installed
- Offer to help set up accounts (Cloudflare, etc.)

**User is an expert:**
- Be straightforward
- Focus on the spec details

**Something isn't working:**
- Read error messages carefully
- Check the Infra guide for common gotchas
- Suggest debugging steps
- Don't guess - investigate

**User wants to customize:**
- Build the base spec first
- Then help with modifications
- Keep customizations separate from core spec
